Fun With JavaScript Loops

Here's a bunch of exercises which can be solved using JavaScript loops. The two types of loops you'll use are for loops and while loops. A loop is a type of control flow, like an if statement. Control flow instructions are special, because they give the computer more information about the order in which normal instructions (like variable assignment or function calls) should be executed. In particular, loops tell the computer to repeatedly execute some set of instructions until a condition is met. This is true of both kinds of loops and, in fact, any problem that could be solved with a for loop could be solved with a while loop instead (though it might not be obvious how).

English has looping structures in it, too, even though you might not think of them that way. Think of a recipe for cake that instructs you to "stir the batter until it is smooth." This is a lot like a while loop: 


    while (batter is not smooth) {
        stir;
    }

Another example could be "pick up all that stuff off the floor" interpreted as a for loop. 


    var count = number of things on the floor;
    for (var i = 1; i <= count; i++) {
        pick up thing #i
    }

Exercise 4: Markov Part I

This is a two-part exercise in which you will write a Markov Text generator. A Markov Text generator takes in a document (a sequence of words, in this case a paragraph from the Wikipedia article on George Washington) and then generates a new document with similar statistical properties. This algorithm is very similar to the one used in modern devices to predict the next word which will be typed based on the one that came before it. The results of basic Markov Text generation are often humorous and word-salad like, in that every pair of words makes sense together, but as a whole the phrases meander nonsensically.

The algorithm is named for Andrey Markov, a Russian mathematician from the 1800's who produced a great deal of work around processes like Markov Text generation. His research is widely used in fields like machine learning, behavior prediction, and computational linguistics.

To get started, we'll first need to know which pairs of words are possible. The idea of Markov Text generation is to always pick a next word that completes a pair we've already seen. For example, if the input document was "I hate that I like you," we know that the word "I" can be followed by either "hate" or "like", but not "you". Exercise 5: Markov Part II

So now that you know how to find the possible subsequents for a single word, you'll need to find them for every word in the input! Markov Text generation works by picking a starting word, and then randomly choosing the word to follow it from its possible subsequents. Then doing the same with that word, and so on, until enough words of Markov Text have been generated.

To do this, you'll first need an association between words and all the words that follow them. For example, if the input document was "I know I want to go to the store, but I don't know what I want to buy," the association would be 


    I      -> know, want, don't, want
    know   -> I, what
    want   -> to, to
    to     -> go, the
    go     -> to
    the    -> store
    store  -> but
    but    -> I
    don't  -> know
    what   -> I
    should -> buy

Notice that following "I" the word "want" is represented twice. This is because "I want" shows up twice in the original document, so "want" is twice as likely to follow "I" as the other two in the list. Starting with "I" and picking randomly from the associations, we could for example generate "I don't know I want to go"

In JavaScript, we store associations using objects. For example, let's say the first word we see is "I". To start off the association, we'd write 


    var association = {};
    association['I'] = [];

This creates an object called association, and it starts off empty. Then we associate "I" with an empty list, because we don't yet know what could follow "I." Then if the next word is "want", we need to add that to the association. I do this using the push method of a JavaScript array. At the end of this line's execution the value of association is { 'I': ['want'] } 


    association['I'].push('want')

Once the association is completed for every word, the Markov Text can be generated as discussed before!
